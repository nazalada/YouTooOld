# Пошаговая инструкция: сборка для Watch Series 3 (watchOS 8) на твоём Mac

Цель: один раз настроить GitHub, чтобы потом спокойно собирать приложение для Apple Watch Series 3 в Xcode на твоём Mac (M2, Xcode 26).

---

## Часть 1. GitHub (один раз)

### Шаг 1. Установи Git (если ещё нет)

- Открой **Терминал** (Программы → Утилиты → Терминал).
- Введи: `git --version` и нажми Enter.
- Если команда не найдена — установи Xcode Command Line Tools: в Терминале введи `xcode-select --install` и дождись установки.

### Шаг 2. Зарегистрируйся на GitHub

- Зайди на [github.com](https://github.com).
- Нажми **Sign up** и создай аккаунт (логин, почта, пароль).

### Шаг 3. Создай новый репозиторий на GitHub

- В правом верхнем углу нажми **+** → **New repository**.
- **Repository name:** например `YouTooOld`.
- Оставь **Public**. Не ставь галочки «Add a README» и т.п.
- Нажми **Create repository**.

### Шаг 4. Открой папку проекта в Терминале

- В Терминале введи (подставь свой путь к папке, где лежит проект):

```bash
cd /Users/naza.prod/Documents/YouTooOld
```

- Нажми Enter.

### Шаг 5. Инициализируй Git и первый коммит (если проект ещё не под Git)

Проверь, есть ли уже Git:

```bash
git status
```

- Если пишет «not a git repository» — выполни по порядку:

```bash
git init
git add .
git commit -m "First commit"
```

- Если `git status` уже показывает файлы и ветки — значит репозиторий есть, этот шаг пропускай.

### Шаг 6. Привяжи проект к GitHub и отправь код

- На странице своего нового репозитория на GitHub нажми зелёную кнопку **Code**.
- Скопируй адрес репозитория (HTTPS), например:  
  `https://github.com/ТВОЙ_ЛОГИН/YouTooOld.git`

В Терминале (всё ещё в папке проекта) выполни по очереди (подставь свой адрес):

```bash
git remote add origin https://github.com/ТВОЙ_ЛОГИН/YouTooOld.git
git branch -M main
git push -u origin main
```

- **Важно:** GitHub не принимает обычный пароль. Нужен **токен** (Personal Access Token). Как создать — см. блок ниже «Токен для git push».

После этого код проекта будет на GitHub.

---

## Токен для git push (обязательно)

GitHub не принимает твой пароль при `git push`. Нужен **Personal Access Token**.

### Как создать токен

1. Зайди на [github.com](https://github.com) и войди в аккаунт.
2. Справа вверху нажми на **свою аватарку** → **Settings**.
3. В левом меню внизу: **Developer settings**.
4. Слева: **Personal access tokens** → **Tokens (classic)**.
5. Нажми **Generate new token** → **Generate new token (classic)**.
6. **Note:** например `YouTooOld Mac`.
7. **Expiration:** выбери срок (например 90 days или No expiration).
8. Отметь галочки:
   - **repo** (полный доступ к репозиториям);
   - **workflow** (обновление GitHub Actions workflow).
9. Внизу нажми **Generate token**.
10. **Скопируй токен** (длинная строка типа `ghp_xxxxxxxxxxxx`) и сохрани в надёжное место. Потом его не покажут снова.

### Как использовать токен при push

В Терминале выполни:

```bash
cd /Users/naza.prod/Documents/YouTooOld
git push -u origin main
```

Когда попросит:
- **Username:** твой логин GitHub (например `nazalada`).
- **Password:** вставь **токен** (не пароль от аккаунта).

Если уже вводил неправильный пароль, система могла его запомнить. Сбрось его:

- **Mac:** Программы → **Связка ключей** (Keychain Access) → поиск по `github` → удали запись для `github.com`.

Потом снова выполни `git push` и при запросе пароля введи **токен**.

## Часть 2. Сборка storyboard в облаке (один раз, потом при изменении storyboard)

### Шаг 7. Запусти сборку storyboard на GitHub

- Открой в браузере свой репозиторий:  
  `https://github.com/ТВОЙ_ЛОГИН/YouTooOld`
- Перейди на вкладку **Actions** (сверху).
- Слева в списке выбери workflow: **«Build WatchKit storyboard for watchOS 8»**.
- Справа нажми **Run workflow** → ещё раз **Run workflow** (зелёная кнопка).

### Шаг 8. Дождись зелёной галочки

- Появится запущенный «run» (жёлтый кружок «In progress»).
- Подожди 3–7 минут. Когда всё пройдёт успешно, кружок станет зелёным.

Если появится красный крестик — открой этот run, посмотри, на каком шаге ошибка (Build, Extract, Update project, Commit), и напиши мне текст ошибки — подскажу, что поправить.

### Шаг 9. Забери изменения к себе на Mac

- Снова открой **Терминал**.
- Перейди в папку проекта:

```bash
cd /Users/naza.prod/Documents/YouTooOld
```

- Выполни:

```bash
git pull
```

- После этого в папке `YouTooOld/YouTooOld WatchKit App/` появится (или обновится) папка **Interface.storyboardc** — это уже скомпилированный storyboard. Его и будет использовать Xcode 26.

## Часть 3. Сборка приложения в Xcode на твоём Mac

### Шаг 10. Открой проект в Xcode

- Открой Xcode.
- **File → Open** → выбери папку с проектом (там, где лежит `YouTooOld.xcodeproj`), открой именно **YouTooOld.xcodeproj**.

### Шаг 11. Выбери схему и устройство

- Сверху в Xcode выбери схему: **YouTooOld**.
- Рядом выбери устройство: **My Mac** (для симулятора) или **iPhone …** (если ставишь на телефон, с которого потом пойдёт установка на часы).

### Шаг 12. Собери проект

- Нажми **Product → Build** (или Cmd+B).
- Сборка должна пройти без ошибки про storyboard. В приложение попадёт готовый `Interface.storyboardc`, поэтому Xcode 26 не будет вызывать старый компилятор storyboard.

### Шаг 13. Установка на часы (если нужно)

- Подключи iPhone, на котором сопряжены часы.
- Выбери этот iPhone как устройство и нажми **Run** (или Cmd+R).
- Приложение установится на телефон и при необходимости на часы (через приложение «Часы» на iPhone).

## Краткая шпаргалка потом

| Что делаешь | Действие |
|-------------|----------|
| Обычная работа в Xcode | Просто собираешь **YouTooOld** (Cmd+B / Run). |
| Поменял экраны/кнопки в storyboard | В GitHub: **Actions** → **Build WatchKit storyboard for watchOS 8** → **Run workflow** → дождаться зелёной галочки → у себя в Терминале: `git pull` → снова собирать в Xcode. |
| Клонировал проект на другой компьютер | Сделай `git pull`, потом открой проект в Xcode и собери. |

## Если что-то пошло не так

- **«git: command not found»** — установи Xcode Command Line Tools: в Терминале `xcode-select --install`.
- **Ошибка при `git push`** — проверь логин/пароль (или токен) GitHub и что репозиторий создан и адрес `origin` правильный.
- **Workflow в GitHub упал с ошибкой** — открой упавший run, посмотри красный шаг и текст ошибки; пришли его — разберём.
- **В Xcode при сборке снова ошибка про storyboard** — убедись, что после успешного workflow ты сделал `git pull` и в папке `YouTooOld WatchKit App` есть папка **Interface.storyboardc**.

Если напишешь, на каком шаге застрял и что именно видишь на экране (или текст ошибки), дам точные действия под твой случай.
